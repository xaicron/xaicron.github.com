Win32::APIとは

  • その名の通り、Windowsを扱うためのAPI
  • Win32::APIはPerlのバインディング
  • Active PerlおよびStrawberry Perlには標準で付属
  • これを使えばDLLの関数をPerlから呼び出すことが可能に

Win32::APIとは

use Win32::API;
my $MessageBox = Win32::API->new(
    "user32",      # 呼び出し元DLL名
    "MessageBoxA", # 呼び出す関数
    "NPPN",        # 関数の引数
    "N"            # 戻り値
);
$MessageBox->Call(0, "Hello, Casual!\n", "Message", 4096);

こんな感じで関数を呼び出してつかう

Win32::Unicode

Win32::Unicode

  • WindowsでUnicodeなファイルとかを簡単に扱えるWin32::APIのラッパー
  • Perlで普通にファイルを扱うのとほとんど同じ感覚で使える
  • Perl内では常にflagged utf8で扱うのでモダンな気持ちを味わえる
  • ついでにコマンドプロンプトにもそのままutf8な文字を出力しちゃう
  • 兎に角、関数をエクスポート

Win32::Unicode::File

IO::Fileっぽく

use strict;
use warnings;
use utf8;
use Win32::Unicode::File;

my $wfile = Win32::Unicode::File->new;
$wfile->open('>', '森鷗外.txt'); # Unicodeファイル
$wfile->binmode(':encoding(cp932)');
$wfile->write("こんにちわ Unicode\n");
$wfile->close;

Win32::Unicode::File

binmode $fh, ':encoding(cp932)';
print {$fh} "こんにちわ Unicode\n";
close $fh;

Win32::Unicode::Native

Win32::Unicode::Native

  • useするだけでほぼ完全に普通のPerlと同じように扱える
  • 新しいことを覚えたくないあなたにぴったり
  • CORE::GLOBALは書き換えず、あくまでエクスポート
  • なので有効範囲はuseしたところのみ
  • 例外として、print系は変更範囲がプログラム全体に及ぶ

Win32::Unicode::Native

エクスポートされる関数
